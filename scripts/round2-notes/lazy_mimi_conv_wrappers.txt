Side-effect bug fix: while exercising the wrappers I uncovered a pre-existing bug in fuel-core/src/lazy_mimi_conv.rs `pad_last_1d` Replicate path: it called `repeat(Shape::from_dims(&[b, c, left]))` which `repeat` interprets as per-axis repeat counts — so when c > 1 the channel dim multiplied (e.g. 2-channel input padded with Replicate produced 4 channels, triggering concat shape mismatch). The original sub-port 1 test only used c=1 so the latent bug never surfaced. Fixed to `repeat(Shape::from_dims(&[1, 1, left]))`.
