## Notes
Single-file module at fuel-core/src/lazy_wuerstchen.rs (1328 LOC). Module + lib.rs registration were already present in the worktree from a prior session, but two of the five required tests were failing with "concat: tensors must live on the same graph". Root cause: text-embedding tensors created via `LazyTensor::from_f32` (a new graph) were being concatenated with model-internal tensors anchored to a different graph inside `apply_attn_block`'s self-attention concat. Fixes:
1. `diffnext_forward_shape_finite_tiny`: switched the text tensor from `LazyTensor::from_f32(...)` to `xs.const_f32_like(...)` so it shares the input xs's graph (matching the pattern the passing prior test already used).
2. `generate(...)` used `LazyTensor::randn` to seed Stage C / Stage B noise — also a fresh graph. Replaced with a new `noise_on_graph(anchor, shape, seed)` helper that anchors a deterministic small-noise constant on `text_embed`'s graph via `const_f32_like`. The denoising loop is deterministic in this v1 port (no DDPM noise added), so reproducible pseudo-random initialization is the natural fit.

All 5 tests pass on CPU. Surface matches spec: `WuerstchenConfig` (with `tiny()`), `PaellaVqModel` (decode), `PriorModel` (forward), `DiffNextModel` (forward), `generate(...)`.
