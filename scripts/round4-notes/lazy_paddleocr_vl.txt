PaddleOCR-VL composition layer wiring text + vision sub-ports. Pipeline: aspect_ratio_chooser picks a (rows, cols) tile grid from the (C, H, W) image; partition_image carves it host-side into row-major tiles; each tile gets nearest-neighbor resized to the encoder's (image_size, image_size) working resolution; the stacked tile tensor flows through PaddleOcrVlVisionModel for projected vision features; host-side run-segmentation builds the combined text+vision embedding by alternately slicing text_embeds (anchored on the vision graph via embed_tokens_anchored) and vision_embeds; the result feeds PaddleOcrVlTextModel::forward_embeds. Key gotcha: text and vision streams must share a graph -- the first attempt used Device::cpu()-anchored embed_tokens and concat panicked with "tensors must live on the same graph"; switching to vision_embeds.embed_tokens_anchored fixed it. Validates that #placeholders == #vision_tokens at build time. File path: fuel-core/src/lazy_paddleocr_vl.rs
