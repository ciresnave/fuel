The file fuel-core/src/lazy_nn_optim.rs already existed in the worktree with a complete implementation (LazyVar + LazyOptimizer trait + LazySgd + LazyAdamW + 9 tests covering all spec-required cases). It was not yet wired into fuel-core/src/lib.rs; this session added `pub mod lazy_nn_optim;` alphabetically (between lazy_mpt and lazy_nomic_bert). All 9 tests pass on CPU. Key design notes: LazyVar carries name + shape + Arc<RwLock<Vec<f32>>> host buffer + Arc<RwLock<Option<NodeId>>> last_node; tensor(&anchor) splices the current value as a graph const and records the issued NodeId so backward_step can fetch the gradient via Tensor::from_existing + GradMap::get. step() builds update ops as LazyTensor expressions and realizes everything via realize_many_f32. SGD momentum/weight_decay match torch semantics (g' = g + wd*w, v = mom*v + g', w -= lr*v). AdamW uses decoupled weight decay (w *= (1 - lr*lambda); w -= lr_scale_m * m / (sqrt(v)*sqrt(scale_v) + eps)) — bit-equivalent to the eager impl's fused-scalar form. F32 + CPU realize only for v1; other dtypes/devices deferred together with port-training-augmentations.md.
