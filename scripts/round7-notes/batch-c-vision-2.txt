## Batch C Vision 2 — pattern observation and blocker analysis

**Single migrated binary (resnet)** was already present on the branch before this session — confirmed by reading `fuel-examples/examples/resnet/main.rs` (88+ lines using `fuel::lazy_resnet::{ResNetConfig, ResNetModel, ResNetWeights}` + `MmapedSafetensors::new(&model_file)` + `ResNetWeights::load_from_mmapped(&st, &config)`). I verified it builds clean (`cargo build --example resnet` → Finished after a transient deepseek2.rs `HostBuffer::I8` match-arm fix unrelated to this task, which I reverted before exit).\n\n**Common blocker for the other 9 vision binaries:** the corresponding `lazy_<name>.rs` module ships the architecture (config + weight structs + forward) but does NOT ship a safetensors weight adapter — i.e. there is no `Weights::load_from_mmapped(st, cfg)` impl translating eager `fuel_transformers` safetensors key conventions (timm/`*.in1k`, apple/MobileCLIP, vincent-espitalier/fuel-eva2, lmz/fuel-efficientnet) into the lazy `WeightStorage` tree. Without that, the eager binary's `VarBuilder::from_mmaped_safetensors` → `Module::new(vb, cfg)` path has no lazy equivalent.\n\nWriting one inline per binary is feasible for ResNet (~300 lines for a single linear architecture) but for these is significantly more complex per model: EfficientNet has MBConv-with-SE expand/depthwise/project trees; FastVit has RepMixer/attention stage forks; MobileOne+RepVGG add structural-reparameterization-fold decisions; MobileNetV4 has Conv-vs-Hybrid forks plus 5+ block types; MobileCLIP needs vision+text+projection joint loader plus a pickle/.bin path. Each is a per-architecture loader design task, not mechanical example-binary migration.\n\n**Recommendation for picker on next session:** prioritize shipping `WeightsX::load_from_mmapped()` adapters on these 9 lazy modules (mirroring `lazy_resnet`'s pattern in `fuel-core/src/lazy_resnet.rs:326-361`); then this batch can re-run mechanically against the canonical `resnet/main.rs` template above. Ordering by loader complexity (low → high): efficientvit, hiera, mobileone, repvgg, eva2, mobilenetv4, efficientnet, fastvit, mobileclip.\n\n**Worktree state at exit:** only `fuel-examples/examples/resnet/main.rs` is "modified" — that's just the BOM-stripping from a Read-then-no-write cycle on the already-migrated file; substantive content is unchanged from main.
